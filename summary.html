1. Lower overhead setting a property

Writing a data-bound number property 2000× in a loop. Nitro’s set() is a synchronous JSI call (straight into C++, applied immediately); legacy’s setNumber() is an async bridge round-trip (serialized, queued, applied later).

Nitro Legacy
~17× cheaper per write (~1 µs vs ~17.5 µs). And it understates it — legacy’s number is only the enqueue cost; native still processes the queue afterward. Frequent updates (sliders, gestures, data-driven animation) stay cheap and immediate on nitro.

2. Separate load & display — pre-pay the read

Legacy can only read+display together, so a view always parses the whole file when it mounts. Nitro’s separate RiveFile API lets you do the read earlier — on a previous screen, a splash, or while scrolling ahead — so the view itself just displays. Test: one view of a 2.9 MB file, cold vs pre-loaded.

milliseconds · lower is better

Nitro Legacy
Cold, it’s a wash (~180 ms both) — the 2.9 MB read dominates and both pay it. The win is that nitro can move that read off the critical path: pre-load on a prior screen and the view appears in ~18 ms. Legacy has no file API, so it must read on mount every time. (And once read, the file is reused across views — see #3.)

3. Scales far better across many views

Showing the same file in N views. Nitro shares one parsed RiveFile, so cost stays flat; legacy re-parses and re-stores the file in every view, so both time and memory climb with N.

Nitro Legacy
~30× faster to fill 24 heavy views (130 ms vs ~4 s) and less memory. Legacy fills the grid over ~4 seconds (views pop in one-by-one) and holds a separate copy per view; nitro parses once and shows them all in the same frame.

4. Eager memory release on unmount

Live heap while mounting then unmounting a heavy screen (6× jellyfish) — like navigating into a Rive-heavy screen and back. Nitro frees native resources via dispose() the moment the view unmounts; legacy waits for deferred cleanup.

Nitro Legacy
Nitro snaps back to baseline immediately; legacy drains slowly. On unmount nitro returns to the exact baseline in <1 s; legacy stays ~19 MB high and bleeds down over ~10 s. Great when navigating between screens that each show heavy .riv files.
